### 第 33-35 列：物理延迟特征（3 个）

基于推流式反应器的水力延迟效应。

| 序号 | 列名 | 延迟时间 | 计算方式 | 物理意义 |
|------|------|---------|----------|----------|
| 33 | `COD_in_delayed` | τ ≈ 3.6h | shift(21) | 3.6 小时前的进水 COD |
| 34 | `NH4_in_delayed` | τ ≈ 3.6h | shift(21) | 3.6 小时前的进水氨氮 |
| 35 | `TN_in_delayed` | τ ≈ 3.6h | shift(21) | 3.6 小时前的进水总氮 |

**延迟时间计算**：
```
τ = HRT × 0.6
  = 6.0 h × 0.6
  = 3.6 h
  
转换为数据点数：
points = τ / dt = 3.6 h / (10 min) = 21 个点
```

**物理意义**：
- **推流式反应器**：水流从进水端流到出水端需要时间
- **0.6 系数**：经验值，表示平均停留时间的 60%
- **应用**：当前时刻的出水水质受 τ 时间前的进水影响

**⚠️ 缺失值说明**：
- 前 21 个点的 delayed 特征为 NaN（没有历史数据）
- 这是正常现象，已在数据预处理中删除这些行

### 第 36-39 列：时间特征（4 个周期性编码）

将时间信息编码为 sin/cos周期性特征。

| 序号 | 列名 | 编码公式 | 值域 | 几何意义 |
|------|------|---------|------|----------|
| 36 | `hour_sin` | sin(2π × hour / 24) | [-1, 1] | 小时在单位圆上的 y 坐标 |
| 37 | `hour_cos` | cos(2π × hour / 24) | [-1, 1] | 小时在单位圆上的 x 坐标 |
| 38 | `dow_sin` | sin(2π × day_of_week / 7) | [-1, 1] | 星期在单位圆上的 y 坐标 |
| 39 | `dow_cos` | cos(2π × day_of_week / 7) | [-1, 1] | 星期在单位圆上的 x 坐标 |

**编码示例**：

| 时间 | hour_sin | hour_cos | dow_sin | dow_cos |
|------|----------|----------|---------|---------|
| 00:00 (周一) | 0.000 | 1.000 | 0.000 | 1.000 |
| 06:00 (周三) | 1.000 | 0.000 | 1.000 | 0.000 |
| 12:00 (周五) | 0.000 | -1.000 | -1.000 | 0.000 |
| 18:00 (周日) | -1.000 | 0.000 | 0.000 | -1.000 |
| 23:00 (任意) | -0.259 | 0.966 | - | - |

**为什么用周期性编码？**

❌ **直接用小数的缺点**：
```
hour = 23 和 hour = 0
数值差距 = 23（很大，但实际上它们是相邻的）
```

✅ **sin/cos 编码的优点**：
```
hour = 23: (sin≈-0.26, cos≈0.97)
hour = 0:  (sin=0.00, cos=1.00)
欧氏距离 ≈ 0.26（很小，正确反映相邻关系）
```

**用途**：
- 捕捉日内变化规律（白天 vs 夜晚）
- 捕捉周内变化规律（工作日 vs 周末）
- 帮助神经网络理解时间的循环特性

```python
neg_mask = df_wide['Q_in'] < 0
if neg_mask.any():
    median_val = df_wide['Q_in'].median()
    df_wide.loc[neg_mask, 'Q_in'] = median_val

# 物理范围截断
physical_limits = {
    'Q_in': (0, 5000),
    'COD_in': (0, 2000),
    # ... 其他指标
}
for col, (min_val, max_val) in physical_limits.items():
    df_wide[col] = df_wide[col].clip(min_val, max_val)

# 4-sigma 平滑（温和方式）
for col in ['Q_in', 'COD_in', 'NH4_in']:
    mean_val = df_wide[col].mean()
    std_val = df_wide[col].std()
    lower = mean_val - 4 * std_val
    upper = mean_val + 4 * std_val
    outliers = (df_wide[col] < lower) | (df_wide[col] > upper)
    if outliers.any():
        # 用移动平均替换
        df_wide.loc[outliers, col] = df_wide[col].rolling(window=3, center=True).mean().loc[outliers]
```

**为什么在特征工程之前处理异常值？**

✅ **避免异常值传播**：
```python

NH4_load = Q_in * NH4_in / 1000  # 如果 Q_in 有负值，负荷也会有问题
# 然后才处理 Q_in 的负值 → 太晚了！

# ✅ 正确做法（v3.0）：先处理异常值，再计算负荷
# Step 5: 修正 Q_in 负值
# Step 6: 计算 NH4_load = Q_in * NH4_in / 1000  # 基于已清洗的数据
```

✅ **滚动统计更可靠**：
```python
# 基于已清洗的数据计算滚动统计
NH4_in_mean_1h = rolling_mean(NH4_in_cleaned, window=6)
# 不会因为某个异常值导致整个窗口的均值偏移
```

```python
for col in ['Q_in', 'MLSS', 'DO', 'NO3', 'Temperature', 'Q_air']:
    df_wide[col] = df_wide[col].ffill()

# 删除仍有缺失的行（通常是文件开头）
df_wide = df_wide.dropna().reset_index(drop=True)
```

**构造的特征**：
1. **物理负荷**（2 个）：NH4_load, COD_load（基于已清洗的数据）
2. **HRT**（1 个）：常数 6.0
3. **延迟特征**（21 个）：
   - 滚动统计：18 个（3 个指标 × 3 个窗口 × 2 个统计量）
   - 物理延迟：3 个（shift 21 个点）
4. **时间特征**（4 个）：hour_sin/cos, dow_sin/cos

## ⚠️ 关键假设与不确定性

### 1. HRT 经验值

| 项目 | 内容 |
|------|------|
| **当前值** | 6.0 h |
| **来源** | 典型城市污水厂好氧池经验范围（6-8 h） |
| **影响** | 延迟特征的计算（τ = 3.6h） |
| **风险** | 实际 HRT 可能不同，导致延迟时间不准确 |
| **应对** | MVP 阶段使用经验值，后续从工艺获取准确池容 |
| **更新公式** | HRT = V_reactor / Q_in |

### 2. 进水端代表性

| 项目 | 内容 |
|------|------|
| **假设** | 2 期进水数据可直接用于 1 期模型 |
| **依据** | PRD v6 第 1.2 节：进出水总管共用 |
| **风险** | 2 期进水仪表位置与 1 期路径可能不同 |
| **影响** | 进水水质可能无法完全代表 1 期实际进水 |
| **应对** | MVP 阶段直接使用，如预测偏差大再现场核实管路拓扑 |

### 3. MLVSS 估算（未使用但需了解）

| 项目 | 内容 |
|------|------|
| **现状** | 现场无 MLVSS 在线仪表 |
| **MVP 处理** | 暂不使用 MLVSS，直接用 MLSS |
| **未来方案** | MLVSS = MLSS × 0.70（典型比值） |
| **不确定度** | ±10%（典型城市污水范围 0.65-0.75） |
